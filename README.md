# HibiNote

チーム向けの軽量な作業日報共有サイト。合言葉を知っている人なら誰でも閲覧・投稿できます。
アカウント登録不要、Firebase (Firestore + Anonymous Auth) + GitHub Pages で動作します。

**このサイトを開くだけで、誰でも自分専用の新しいチームを作れます。** Firebaseの知識やアカウント作成は一切不要です。裏側ではリポジトリ管理者が用意した1つのFirebaseプロジェクトを、チームごとに独立した「ルーム」として共有しています。

詳しい仕様は [CLAUDE.md](./CLAUDE.md) を参照してください。

## 使い方(誰でも・チームの管理者)

1. サイト(例: `https://<ユーザー名>.github.io/HibiNote/`)を開く
2. 何もしなくても新しいチームが自動的にできています。表示された**招待リンク**を控えて「はじめる」を押す
3. 合言葉入力画面の「招待リンク・合言葉を確認する」から、そのチームの合言葉を発行する(6桁の数字。このパネルからいつでも確認・コピー・発行し直しができます)
4. **招待リンク**と**合言葉**をチームメンバーに共有する

これだけです。Firebaseプロジェクトの作成やコードの編集は不要です。

## 使い方(チームメンバー)

1. 管理者から共有された**招待リンク**を開く(初回のみ。以後はその端末に接続先が保存されます)
2. 管理者から共有された**合言葉**を入力する
3. 以降はその端末では自動的にログインされます

- ヘッダーの「ログアウト」ボタンで合言葉ロックに戻せます(端末共有時など)
- ヘッダー・ゲート画面の「招待リンク」からいつでもリンクを再取得できます
- ゲート画面の「別のチームに切り替える」を押すと、今のチームから離れて新しいチームを自動的に作れます(既存チームに参加したいだけなら、そのチームの招待リンクを開いてください)
- 投稿は自動で作業コード(`0001`など)が採番されます。ヘッダーの検索窓に数字を入力すると該当の投稿にジャンプでき、`index.html?code=0001` の形式で直リンクも可能です

## セットアップ(このリポジトリを管理している人向け)

サイトの裏側で使うFirebaseプロジェクトを1つ用意し、コードに埋め込みます(利用者側の作業ではありません)。

### 1. Firebaseプロジェクトを作る

1. [Firebaseコンソール](https://console.firebase.google.com/)で新規プロジェクトを作成(無料のSparkプランでOK)
2. 「Firestore Database」を有効化(本番モードでOK、後述のルールを適用)
3. 「Authentication」→ Sign-in method で「匿名」を有効化
4. プロジェクト設定 > 全般 > マイアプリ で `</>`(ウェブ)アプリを追加し、`<script>` タグを使用する を選択して表示される `firebaseConfig` の値を `js/firebase-config.js` に貼り付ける

### 2. Firestoreセキュリティルールを設定

Firebaseコンソールの Firestore Database > ルール に、このリポジトリの `firestore.rules` の内容を貼り付けて公開します。

### 3. GitHub Pagesでデプロイ

このリポジトリの Settings > Pages で、公開ブランチ(例: `main`)とルートディレクトリを指定して有効化します。ビルド手順は不要で、静的ファイルがそのまま配信されます。

以降は誰でもこのURLを開くだけで新しいチームを作って使えます。多数のチームが同じFirebaseプロジェクトの無料枠を共有する点だけ留意してください。

## ファイル構成

```
index.html            画面全体(新規チーム作成/合言葉ゲート/一覧/詳細/投稿・編集)
css/style.css          スタイル
js/firebase-config.js   埋め込みのFirebaseプロジェクト設定(管理者が1回だけ設定)
js/firebase.js          Firebase初期化・匿名認証
js/project.js           ルームIDの生成・保存・招待リンクの発行
js/image.js             画像のcanvas圧縮
js/main.js              アプリ本体のロジック(起動フロー・合言葉の発行検証を含む)
firestore.rules         Firestoreセキュリティルール
```
