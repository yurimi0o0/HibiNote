# HibiNote

チーム向けの軽量な作業日報共有サイト。合言葉を知っている人なら誰でも閲覧・投稿できます。
アカウント登録不要、Firebase (Firestore + Anonymous Auth) + GitHub Pages で動作します。

このサイトはコードに特定のFirebaseプロジェクトを埋め込んでいません。**同じサイトを、無関係な複数のチーム・プロジェクトがそれぞれ自分のFirebaseプロジェクトに接続して使い回せます。** リポジトリをforkする必要はありません。

詳しい仕様は [CLAUDE.md](./CLAUDE.md) を参照してください。

## 使い始め方(チームの管理者)

### 1. 自分のFirebaseプロジェクトを作る

1. [Firebaseコンソール](https://console.firebase.google.com/)で新規プロジェクトを作成(無料のSparkプランでOK)
2. 「Firestore Database」を有効化(本番モードでOK、後述のルールを適用)
3. 「Authentication」→ Sign-in method で「匿名」を有効化
4. プロジェクト設定 > 全般 > マイアプリ で `</>`(ウェブ)アプリを追加し、`<script>` タグを使用する を選択して表示される `firebaseConfig` のコードをコピー

### 2. Firestoreセキュリティルールを設定

Firebaseコンソールの Firestore Database > ルール に、このリポジトリの `firestore.rules` の内容を貼り付けて公開します。

### 3. サイトを開いて設定を貼り付ける

このサイト(例: `https://<ユーザー名>.github.io/HibiNote/`)を開くと、初回は「Firebaseプロジェクトの設定」を貼り付ける画面が出ます。手順1でコピーしたコードをそのまま貼り付けて保存すると、そのブラウザがあなたのFirebaseプロジェクトに接続されます。

保存すると「招待リンク」が発行されます。このリンクを開いた人は、貼り付け作業なしで同じプロジェクトに接続されます。**このリンクをチームメンバーに共有してください。**

### 4. 合言葉を発行する

合言葉入力画面の「管理者用: 合言葉を発行する」から発行します(Node.jsスクリプトの実行は不要)。
ランダムな8桁の数字がその場で発行され、画面に一度だけ表示されるのでその場でコピーしてチームメンバーに個別に共有してください(招待リンクとは別に伝える2つ目の合言葉です)。
新しいチーム・プロジェクトで使い回す際は、同じ画面から発行し直せば古い合言葉は使えなくなります(すでにログイン済みの端末はそのまま利用できます)。

## 使い方(チームメンバー)

- 管理者から共有された**招待リンク**を開く(初回のみ。以後はそのブラウザに接続先が保存されます)
- 管理者から共有された**合言葉**を入力する
- 以降はその端末では自動的にログインされます
- ヘッダーの「ログアウト」ボタンで合言葉ロックに戻せます(端末共有時など)
- ゲート画面の「別のプロジェクトに切り替える」で、接続先のFirebaseプロジェクトごと変更できます
- 投稿は自動で作業コード(`0001`など)が採番されます。ヘッダーの検索窓に数字を入力すると該当の投稿にジャンプでき、`index.html?code=0001` の形式で直リンクも可能です

## デプロイ(このリポジトリを管理している人向け)

このリポジトリの Settings > Pages で、公開ブランチ(例: `main`)とルートディレクトリを指定して有効化します。ビルド手順は不要で、静的ファイルがそのまま配信されます。Firebaseプロジェクトの作成やforkは不要で、利用者ごとにブラウザ側で接続先を切り替えます。

## ファイル構成

```
index.html      画面全体(初回設定/合言葉ゲート/一覧/詳細/投稿・編集)
css/style.css    スタイル
js/firebase.js    Firebaseの遅延初期化・匿名認証
js/project.js     Firebase設定の保存・招待リンクの生成/解析
js/image.js       画像のcanvas圧縮
js/main.js        アプリ本体のロジック(起動フロー・合言葉の発行検証を含む)
firestore.rules   Firestoreセキュリティルール
```
